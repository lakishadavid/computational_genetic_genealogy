What is Git?

Git is a distributed version control system originally developed by Linus Torvalds (creator of Linux) to manage large software projects. In research, Git helps us:

  • Track changes in our code and analysis scripts
  • Collaborate with other researchers without overwriting each other's work
  • Maintain different versions of our analysis pipelines
  • Document when and why changes were made
  • Share our work with the broader scientific community

Installation Instructions

Before installing, note that Git might already be installed on your system. If you're using a Unix-based system (macOS or Linux), Git often comes pre-installed.

Windows

  1. Download the Git installer from Git for Windows
  2. Run the downloaded .exe file
  3. During installation:
    • Accept the default options
    • When prompted about "Adjusting your PATH environment", select "Git from the command line and also from 3rd-party software"
    • For "Configuring the line ending conversions", choose "Checkout Windows-style, commit Unix-style line endings"

macOS

  1. If you have Homebrew installed:
    brew install git
  2. Otherwise, download the installer from Git Downloads

Linux (Ubuntu/Debian)

sudo apt-get update
sudo apt-get install git

Initial Configuration

After installing Git, set up your identity:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Troubleshooting

Common Issues

  1. "git is not recognized as an internal or external command" (Windows)
    • Close and reopen your command prompt
    • Check if Git is added to your PATH
  2. "Permission denied" (macOS/Linux)
    • Add sudo before the installation command
    • Ensure you have administrator privileges

Verification Exercise

To ensure everything is working, run these commands:

git --version
git config --global user.name
git config --global user.email
