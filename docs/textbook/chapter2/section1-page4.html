After installing Git and VSCode, our next step is to download (or "clone" in Git terminology) the course materials from GitHub. In computational research, code sharing platforms like GitHub are essential for distributing analysis pipelines and datasets.

Understanding Git Clone vs. ZIP Download

Why Clone?

  • Version Control: Maintains Git connection for updates and change tracking
  • Complete History: Access the full development history
  • Reproducibility: Aligns with research best practices
  • Real-world Practice: Learn professional workflows

Choosing Where to Clone

Windows (non-WSL2) Users

Choose a location in your Windows filesystem (e.g., Documents/Courses)

WSL2 Users

Recommended: Clone into your WSL2 filesystem (e.g., ~/courses)

Access from Windows: \\wsl$\Ubuntu\home\yourusername\courses

macOS/Linux Users

Choose any location in your home directory (e.g., ~/courses)

Cloning Options

Option 1: Command Line (Recommended)

  1. Open your terminal
  2. Navigate to your chosen directory:
    cd path/to/your/preferred/directory
  3. Clone the repository:
    git clone https://github.com/lakishadavid/computational_genetic_genealogy.git

Option 2: VSCode Interface

  1. Open VSCode (WSL2 users: ensure you're in a WSL2 window)
  2. Open Command Palette (Ctrl+Shift+P or Cmd+Shift+P)
  3. Type "Git: Clone" and select it
  4. Enter the repository URL
  5. Choose your storage location

Verification

  1. Check for the new directory:
    ls computational_genetic_genealogy
  2. Verify Git tracking:
    git status

Troubleshooting

Common Issues

  • "Permission denied" error
    • Check GitHub account and login
    • Verify directory permissions
  • "Could not resolve host" error
    • Check internet connection
    • Verify repository URL
